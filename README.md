# Introduction

## Définition

"JavaScript est un <mark style="color:orange;">langage de script</mark>, <mark style="color:orange;">multi-plateforme et orienté objet</mark>. C'est un langage léger qui doit faire partie d'<mark style="color:orange;">un environnement hôte</mark> (un navigateur web par exemple) pour qu'il puisse être utilisé sur les objets de cet environnement."

### Niveau de permissivité

<mark style="color:orange;">JavaScript est un langage plutôt « libre » comparé au Java</mark>. Il n'est pas nécessaire de déclarer toutes les variables, classes et méthodes.&#x20;

(Il n'est pas nécessaire de savoir si une méthode est publique, privée ou protégée et il n'y a pas d'interfaces à implémenter. Les variables, les paramètres et les valeurs de retour des fonctions ne sont pas explicitement typés.)

```javascript
//déclaration de variable typée en java
double result;

//déclaration de variable non typée en javascript
let result
```

### Rapidité d'éxecution

<mark style="color:orange;">Java</mark> est un langage de programmation utilisant les classes, <mark style="color:orange;">conçus pour être exécuté rapidement et garantir la sûreté du typage</mark>.&#x20;

Cela signifie par exemple qu'<mark style="color:orange;">il n'est pas possible de transformer un entier Java en un objet</mark> ou qu'on ne peut pas accéder à des caractéristiques privées en <mark style="color:orange;">corrompant le bytecode Java</mark>.&#x20;

(Le modèle de classes utilisé par Java signifie qu'un programme n'est constitué que de classes et de méthodes. Cet héritage à base de classes, associé au typage fort font qu'on obtient des structures et des hiérarchies d'objets fortement couplées.)

### Public cible et complexité

Pour les raisons décrites ci-dessus, <mark style="color:orange;">Java peut apparaître comme un langage plus complexe que JavaScript</mark>.

À l'inverse, JavaScript est un descendant de langages plus légers, dynamiquement typés tels que HyperTalk et dBASE. <mark style="color:orange;">Ces langages de scripts visent un public plus large avec une syntaxe plus simple</mark>, des fonctionnalités natives spécialisées et des prérequis minimaux pour pouvoir créer des objets.
