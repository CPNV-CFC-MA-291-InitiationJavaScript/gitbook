# Projet

## Intention pédagogique

Nous arrivons à la fin du module. Il est donc venu le temps de faire un projet permettant de transférer les connaissances théoriques ainsi que les compétences pratiques acquises durant les laboratoires dans un projet pratique. Ce sera également l'occasion de travail en binôme.

## Objectifs opérationnels

Réaliser une application web permettant de s'entraîner au calcul mental. L'application offrira les fonctionnalités suivantes:

* [ ] Zone settings
  * [ ] Permet de modifier la difficulté du jeux.
  * [ ] Saisir son pseudo.
  * [ ] Limiter le temps de la partie.
  * [ ] Limiter le nombre de tentative par question.
  * [ ] Définir le nombre de questions.
  * [ ] Lancer/Stopper la partie.
* [ ] Zone game
  * [ ] Affichera le calcul à résoudre.
  * [ ] &#x20;Permettra à l'utilisateur de transmettre sa tentive de réponse.
  * [ ] Informe l'utilisateur du résultat (juste/faux) ainsi que son score actuel.
* [ ] Zone history
  * [ ] Elle sera vide en début de session.
  * [ ] Chacune des paires question et  réponse, s'ajoute à la liste contenu l'historique.
  * [ ] Dès le début de la partie jusqu'au lancement d'une nouvelle partie, cette liste est affichée.

### Critères techniques

* [ ] L'application soit être totalement adaptée à au moins un type de support, de votre choix (mobile, tablette, desktop).
* [ ] Les scripts doivent être développés en "strict mode".
* [ ] Une fonction - une action précise.
* [ ] Les bonnes pratiques que nous avons vues en classe doivent être respectées.

### Github

* [ ] La structure suivante doit être respectée:
  * [ ] La branche main ne doit pas être modifiée.
  * [ ] La branch develop est la branche de travail et de livraison du projet.
  * [ ] Les commits sont préfixés comme suit:
    * [ ] "feat:" -> pour une nouvelle fonctionnalité
      * [ ] "refactor:" -> pour améliorer du code sans impacter le comportement
      * [ ] "style:" -> pour le css
      * [ ] "fix": pour corriger un erreur dans le code
      * [ ] "docs:" pour toutes modifications impactants la documentation (maquette par exemple)
